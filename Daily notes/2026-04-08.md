## API Contract Discussion with Sarah

Main concern: pagination format. Current response wraps data in `{ items: [], total: N }`, but the mobile team expects `{ data: [], meta: { total, page, perPage } }`. Agreed to add an adapter layer on the BFF side rather than change the core API.

```ts
// BFF adapter
function adaptPagination<T>(raw: LegacyPage<T>): StandardPage<T> {
  return {
    data: raw.items,
    meta: { total: raw.total, page: raw.page, perPage: raw.limit },
  };
}
```
